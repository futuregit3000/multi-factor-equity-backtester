# GitHub Publishing Checklist

1. Create a new public repository named:
   `multi-factor-equity-backtester`
2. Do not initialize it with another README or license.
3. Upload the contents of this package, not the enclosing ZIP file.
4. Confirm that the repository root displays `README.md`.
5. Open the notebook on GitHub and confirm that charts and tables render.
6. Confirm no personal email address is stored in the notebook.
7. Add a short repository description:
   `Point-in-time multi-factor equity backtester using SEC, FRED, and market data.`
8. Add topics:
   `quantitative-finance`, `python`, `backtesting`, `factor-investing`,
   `portfolio-management`, `sec-edgar`
9. Pin the repository to your GitHub profile.
10. Add the repository link to your resume only after you can explain the
    methodology and limitations.
